在debian上安装环宇飞扬的摄像头

Oct 25th, 2005

系统: Duron 900/VIA KT133/debian sid/kernel 2.6.8-2-686

早就知道z-star usb的摄像头有linux驱动了.但是一直嫌麻烦,没去装.

今天又想起来,试着装了下 还是很简单的嘛! 嘿嘿.

首先先lsusb看下我这个摄像头:

$sudo lsusb -v

us 001 Device 002: ID 0ac8:301b Z-Star Microelectronics Corp. Sansun SN-510 WebCam [hv713d]
Device Descriptor:
bLength 18
bDescriptorType 1
bcdUSB 1.10
bDeviceClass 255 Vendor Specific Class
bDeviceSubClass 0
bDeviceProtocol 0
bMaxPacketSize0 8
idVendor 0x0ac8 Z-Star Microelectronics Corp.
idProduct 0x301b Sansun SN-510 WebCam [hv713d]

………………………..

恩 剩下的就不列出来了.

然后,看看我的内核里面有没有videodev模块:

$modprobe -l|grep videodev

/lib/modules/2.6.8-2-686/kernel/drivers/media/video/videodev.ko

可以看出来我的内核里面已经包含了videodev模块,如果没有这个模块那就需要重新编译内核啦.

具体怎么编译内核在这里不细说了,网上能搜到好多.

再然后要看下自己的系统里面带不带kernel-headers

如果不带那就要自己装个喽(例如我这个):

$uname -r #先看看自己的内核版本.

2.6.8-2-686
$sudo aptitude install kernel-headers-2.6.8-2-686 #安装对应版本的kernel-headers喽

这样几个前提条件都具备了,让我们开工编译摄像头的驱动.

首先去 http://mxhaard.free.fr/download.html 这里下载 spca5xx-20050601.tar.gz [当前最新]

然后解压开

$sudo tar zxvf spca5xx-20050601.tar.gz

进入解压后生成的目录:

$cd spca5xx-20050601

恩 直接make:

$sudo make

$sudo make install

如果这两项没提示错误 继续:

$sudo modprobe videodev spca5xx

恩 没提示错误的话, 恭喜! 驱动安装成功了~

恩 然后是创建设备文件:

mknod /dev/video c 81 0

chmod 777 /dev/video #开始的时候权限设为744了 结果gqcam说找不到设备!

最后 咱们拍几张照片,看看效果:

aptitude install gqcam

$gqcam

好啦 看到图像没? 赶快拍下来吧~~
